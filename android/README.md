# Hermes Agent — Android/Termux Deployment

Native Termux deployment for aarch64 Android phones. No proot, no emulation.

## The Problem

Termux uses Android's **Bionic libc** (not glibc), so manylinux wheels from PyPI
don't work. Native C/Rust extensions must be compiled specifically for Termux.

**Problematic packages** (14 native extensions):
- **Rust**: `pydantic-core`, `jiter`, `cryptography`
- **C**: `cffi`, `aiohttp`, `PyYAML`, `MarkupSafe`, `msgpack`

## Solution

Cross-compile inside a `termux/termux-docker:aarch64` container on your Mac
(native arm64 — no emulation needed on Apple Silicon), producing
`android_24_arm64_v8a` wheels that work directly in Termux.

## Quick Start

```bash
# 1. Build native wheels (runs in Docker, ~3 min on M-series Mac)
./android/build.sh

# 2. Deploy to phone over SSH
./android/deploy.sh 192.168.1.42 8022 yourpassword
```

## Manual Install (on phone)

```bash
# In Termux on the phone:
pkg install python ripgrep openssh git

# Clone or copy hermes-agent to the phone, then:
cd ~/hermes-agent

# Option A: use prebuilt wheels (faster)
bash android/install.sh

# Option B: build native wheels on-device (no remote machine needed)
bash android/install.sh --build-native
```

## Phone Setup (Termux prerequisites)

```bash
# Install Termux from F-Droid (NOT Google Play — the Play version is outdated)
# https://f-droid.org/packages/com.termux/

# In Termux:
pkg update && pkg upgrade
pkg install openssh python ripgrep

# Set password for SSH access:
passwd

# Start SSH server (default port 8022):
sshd

# Find your IP:
ifconfig wlan0 | grep 'inet '
```

## Running Gateway as a Termux Service (runit)

`hermes gateway install` only supports systemd/launchd today. On Termux, use `termux-services`.

```bash
# 1) Install service manager
pkg install termux-services
source $PREFIX/etc/profile.d/start-services.sh

# 2) Create Hermes gateway service
mkdir -p $PREFIX/var/service/hermes-gateway/log

cat > $PREFIX/var/service/hermes-gateway/run <<'SH'
#!/data/data/com.termux/files/usr/bin/sh
exec 2>&1
cd "$HOME"
exec hermes gateway run
SH

cat > $PREFIX/var/service/hermes-gateway/log/run <<'SH'
#!/data/data/com.termux/files/usr/bin/sh
pwd=${PWD%/*}
service=${pwd##*/}
mkdir -p "$LOGDIR/sv/$service"
exec svlogd -tt "$LOGDIR/sv/$service"
SH

chmod 700 $PREFIX/var/service/hermes-gateway/run
chmod 700 $PREFIX/var/service/hermes-gateway/log/run

# 3) Start and manage
sv up hermes-gateway
sv status hermes-gateway
sv restart hermes-gateway
sv down hermes-gateway

# Logs
# (gateway logs)
tail -f $PREFIX/var/log/sv/hermes-gateway/current
# (WhatsApp bridge logs)
tail -f ~/.hermes/whatsapp/bridge.log
```

## Files

| File | Purpose |
|------|---------|
| `build.sh` | Build native wheels in Termux Docker |
| `install.sh` | Install on phone (run in Termux) |
| `deploy.sh` | Deploy to phone over SSH (run on Mac) |
| `wheels/` | Pre-built `.whl` files (generated by build.sh) |

## Updating

When hermes-agent deps change:

1. Update pinned versions in `build.sh` (NATIVE_PKGS) and `install.sh` (PURE_DEPS)
2. Run `./android/build.sh` to rebuild wheels
3. Run `./android/deploy.sh` or `./android/install.sh --upgrade` on phone

## Verified Environment

- Termux Python 3.13.12
- Rust 1.94.1
- Android API 24+ (Android 7.0+)
- Docker `termux/termux-docker:aarch64`
